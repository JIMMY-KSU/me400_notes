Like merge sort, the default sorting algorithm used by np.sort is \(O(n \log n)\). Suppose you used np.sort to sort 100 different arrays each having 100,000 numbers, and the average time per sort was about 0.01 s. If you did the same thing for arrays with 1,000,000 numbers each, about how long do you think it would take on the average?

HINT 1: Remember that \(\log_b x^a = a \log_b x\), e.g., \(\log_b 10^3 = 3 \log_b 10\).

HINT 2: We use order as an engineering metric by assuming the time is proportional to whatever is inside the \(O\). If the method is \(O(n)\), then \(t \propto n\) or \(t = an\) for some (unknown) constant \(a\).
