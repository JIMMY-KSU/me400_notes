Lecture 16 - Modules and Packages¶

Python modules offer a unique way to organize your code, especially as it evolves into several functions. Moreover, any of the external packages you’ve already used, like NumPy and Matplotlib, are organized as modules within larger packages. By the end of this lesson, you’ll understand how to organize your create and use your own modules and packages.

Objectives¶

By the end of this lesson, you should be able to

  • Produce a module of functions
  • Set appropriate paths in order to import modules you create

Any time remaining can be spent on Exam questions.

Creating Modules¶

We’ve been importing modules like NumPy since the beginning:

In [1]:

import numpy
type(numpy)

Out[1]:

module

Modules are easy to create: they can be as simple as a single Python script.

Consider these possible ways to quantify a vector \(\mathbf{e}\) that represents errors:

  1. mean, absolute error, \(\sum^n_i |e_i|/n\)
  2. root-mean-square error, \(\sqrt{\sum^n_i e_i^2/n}\)
  3. maximum, absolute error, \(\max(|e_i|) \, \forall i \in [1, n]\)

If \(\mathbf{e}\) is a sequential type, the following are possible implementations:

def mean_abs_error(e):
    """Mean, absolute error."""
    v = 0
    for i in range(len(e)) :
        v += abs(e[i])
    return v/len(e)

def rms_error(e) :
    """Root-mean-square error."""
    v = 0
    for i in range(len(e)) :
        v += e[i]**2
    return (v/len(e))**0.5

def max_abs_error(e) :
    """Maximum, absolute error."""
    v = 0
    for i in range(len(e)) :
        if abs(e[i]) > v:
            v = abs(e[i])
    return v

Functions can be defined almost anywhere in any Python program.

This can be very convenient but opens the door to copy and paste.

When importing modules directly (e.g., import error_metrics), must use the module name to access its functions (e.g., error_metrics.mean_abs_error).

Alternative: use abbreviation like we do for NumPy:

In [4]:

import error_metrics as em
em.mean_abs_error(e)

Out[4]:

0.15000000000000002

Like other names in Python, moduled can be given new names:

In [5]:

em2 = em
em2.mean_abs_error(e)

Out[5]:

0.15000000000000002

Living dangerously: we can simply import everything via

In [6]:

from error_metrics import *
mean_abs_error(e)

Out[6]:

0.15000000000000002

That’s okay, but what about

In [7]:

from math import *
from error_metrics import *
print(pi)

1.618033988749895
“When you use the *, careful you must be, for the * uses you back.” - Said probably not Yoda

Final alternative: import individial functions or attributes (values like pi) via

In [8]:

from error_metrics import mean_abs_error, rms_error
e = [0.1, -0.2, 0.25, -0.05] # why define again?
mean_abs_error(e)

Out[8]:

0.15000000000000002

Functions (or attributes like pi) can be imported selectively via

In [9]:

from error_metrics import rms_error as re
from error_metrics import pi as not_the_real_pi
print(not_the_real_pi)

1.618033988749895

but a statement like

import error_metrics.rms_error as rms_error

will fail.

Running Modules as Scripts¶

Every Python script can be imported as a module (that includes your homework submissions!)

Almost all statements in the script are executed: - variable assignments (like the errant \(pi\)) - function calls - other imports - miscellaneous code for showing/testing

Selective imports can help, but sometimes the extra code is for quick function testing.

An alternative: place if __name__ == "__main__": at the bottom of the file:

if __name__ == "__main__" :
    e = [0.1, 0.2, 0.3]
    print(mean_abs_error(e))
    print(rms_error(e))
    print(max_abs_error(e))

Running this leads to

0.2
0.374165738677
0.3

Show also that import error_metrics does not lead to the print statements when the if __name__ line is included.

Packages¶

Modules collect functions, etc. Packages collect modules.

Example: 1. Make a new directory called my_package and place error_metrics.py in it.
2. In that same directory, create an empty file named __init__.py.

Once that’s done, the following is possible from anywhere:
